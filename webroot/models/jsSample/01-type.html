Number类型
1. 相等运算符
JavaScript在设计时，有两种比较运算符：
(1)第一种是相等操作符(==)
  它会自动转换数据类型再比较，很多时候，会得到非常诡异的结果；
(2)第二种是全等操作符(===)
  它不会自动转换数据类型，如果数据类型不一致，返回false，如果一致，再比较。
由于JavaScript这个设计缺陷，不要使用==比较，始终坚持使用===比较。
2. NaN
NaN(Not a Number)是个特殊的Number与所有其他值都不相等，包括它自己；
唯一能判断NaN的方法是通过isNaN()函数。

5. 无穷大Infinity和无穷小-Infinity
isFinite(number)函数:
	如果 number 是有限数字（或可转换为有限数字），
	那么返回 true。否则，如果 number 是 NaN（非数字），
	或者是正、负无穷大的数，则返回 false。
6.Number()函数的转换规则
如果是Boolean 值，true 和false 将分别被转换为1 和0。
如果是数字值，只是简单的传入和返回。
如果是null 值，返回0。
如果是undefined，返回NaN。
如果是字符串，遵循下列规则：
	1> 如果字符串中只包含数字（包括前面带正号或负号的情况），
  则将其转换为十进制数值，即"1"会变成1，"123"会变成123，
  "2.14e5"会变成214000，而"011"会变成11（注意：前导的零被忽略了）；
	2> 如果字符串中包含有效的浮点格式，
  如"1.1"，则将其转换为对应的浮点数值（同样，也会忽略前导零）；
	3>如果字符串中包含有效的十六进制格式，例如"0xf"，
  则将其转换为相同大小的十进制整数值；
	4>如果字符串是空的（不包含任何字符），则将其转换为0；
	5>如果字符串中包含除上述格式之外的字符，则将其转换为NaN。
	6>如果是对象，则调用对象的valueOf()方法，
  然后依照前面的规则转换返回的值。
  如果转换的结果是NaN，则调用对象的toString()方法，
  然后再次依照前面的规则转换返回的字符串值。
7. parseInt()函数
专门用于把字符串转换成整数。
由于Number()函数在转换字符串时比较复杂而且不够合理，
因此在处理整数的时候更常用的是parseInt()函数。
  1> 用parseInt()转换空字符串会返回NaN（Number()对空字符返回0）
  2> "1234blue"会被转换为1234，因为"blue"会被完全忽略
  3> "22.5"会被转换为22，因为小数点并不是有效的数字字符。
  4> 在使用parseInt()解析像八进制字面量的字符串时，
 ECMAScript 3 和5 存在分歧。例如：
 //ECMAScript 3 认为是56（八进制），ECMAScript 5 认为是70（十进制）
  5> 提供第二个参数,指定基数：parseInt("71", 8)转换为57
8. parseFloat()函数
  1> 字符串中的第一个小数点是有效的，而第二个小数点就是无效的
     "22.34.5"将会被转换为22.34
  2> parseFloat()只解析十进制值
  	 "0x71"被转换成0
  3> 没有用第二个参数指定基数的用法
7. toString()方法
可以传递一个参数：输出数值的基数
8. toFixed()方法
按照指定的小数位返回数值的字符串表示，例如：
var num = 10;
num.toFixed(2) --> "10.00"
在给toFixed()传入0 的情况下，
IE8 及之前版本不能正确舍入范围在{(-0.94,-0.5],[0.5,0.94)}之间的值。
对于这个范围内的值，IE 会返回0，而不是-1 或1；
9. toExponential()方法
返回以指数表示法（也称e 表示法）表示的数值的字符串形式。
var num = 10;
num.toExponential(1)) --> "1.0e+1"
10. toPrecision()方法
可能会返回固定大小（fixed）格式，也可能返回指数（exponential）格式；
具体规则是看哪种格式最合适。
var num = 99;
num.toPrecision(1) --> "1e+2"
num.toPrecision(2) --> "99"
num.toPrecision(3) --> "99.0"
null和undefined
null和undefined基本是同义的，只有一些细微的差别。
1. null表示"没有对象"，即该处不应该有值。典型用法是：
（1） 作为函数的参数，表示该函数的参数不是对象。
（2） 作为对象原型链的终点。?待研究
2. undefined表示"缺少值"，就是此处应该有一个值，但是还没有定义。典型用法是：
（1）变量被声明了，但没有赋值时，就等于undefined。
（2) 调用函数时，应该提供的参数没有提供，该参数等于undefined。
（3）对象没有赋值的属性，该属性的值为undefined。
（4）函数没有返回值时，默认返回undefined。

JavaScript内置对象
JavaScript 提供多个内置对象，比如 String、Date、Array、Math 等等。
直接创建对象
var person = {};
person.firstname="Bill";
person.lastname="Gates";
person.age=56;
person.eyecolor="blue";
使用函数构造对象
function person(firstname,lastname,age,eyecolor){
	this.firstname=firstname;
	this.lastname=lastname;
	this.age=age;
	this.eyecolor=eyecolor;
}

var bill = new person("Bill","Gates",56,"blue");
