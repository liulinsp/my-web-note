ECMAScript
Sun(现在的Oracle)公司持有着“Java”和“JavaScript”的商标。这就让微软不得不把自己的JavaScript方言称
之为“JScript”。然后，在这门语言被标准化的时候，就必须使用一个与二者都不同的名字。“ECMAScript”
就这样诞生了，这个名字的来由是因为执行标准化的组织是Ecma国际。通常来说，术语“ECMAScript”和
“JavaScript”指的是同一个东西。但如果把JavaScript看成是“Mozilla或其他组织的ECMAScript实现”，那么
ECMAScript就是实现JavaScript所依据的标准。

ECMA-262
Ecma国际 (一个标准化组织)创建了ECMA-262规范,这个规范就是ECMAScript语言的官方标准。
历史版本
ECMAScript 1 【1997】
1997年6月发布，本质上与javascript 1.1 相同——只不过只不过删除了所有针对浏览器的代码并作了一些较小
的改动：ECMAScript要求支持Unicode标准，而且对象也变成了平台无关的

ECMAScript 2 【1998】
1998年6月发布，主要是编辑加工的结果。这一版的内容更新是为了与ISO/IEC-16262保持严格一致，没有作
任何新增、修改或删节处理。因此，一般不使用第2版来衡量ECMAScript实现的兼容性。

ECMAScript 3 【1999】
1999年12月发布，是对ECMAScript标准第一次真正的修改。
1. 新增功能：
* 正则表达式
* 新控制语句
* try-catch异常处理
2. 修改功能：
* 字符处理
* 错误定义
* 数值输出等内容
从各方面综合来看，第3版标志着ECMAScript成为了一门真正的编程语言。

ECMAScript 4 【2008】
于2008年7月发布前被废弃……命运坎坷。
没有公布的ECMAScript第四版被编入第六版的开发。

ECMAScript 5 【2009】
2009年12月发布，该版本力求澄清第3版中的歧义，并添加了新的功能。新功能包括：
* 原生JSON对象
* 继承的方法
* 高级属性的定义
* 严格模式

ECMAScript 6 【2015】
2015年6月发布，ES6增添了许多必要的特性，例如：
* modules(模块化), 
* class declarations(类的声明), 
* lexical block scoping(块级作用域), 
* iterators(迭代器) and generators(生成器), 
* promises(异步编程的promise), 
* destructuring patterns(解构), 
* proper tail calls(尾调用的优化). 

ECMAScript内置库还增加了数据抽象的支持，包括：
* Map
* Set
* arrays of binary numeric values(二进制数值数组)
* Unicode supplemental characters in strings and regular expressions(字符串和正则对Unicode的支持)
 
ECMAScript 2016 【第7版】
第一个以年度命名的版本，并开始在GitHub上进行开发，新增功能有：
* 幂运算符
* Array.prototype.includes 方法

ECMAScript 2017 【第8版】
 新增异步功能：
* Async Functions 异步函数
* Shared Memory 共享内存
* Atomics 原子操作

 新增对象静态方法
* Object.values
* Object.entries
* Object.getOwnPropertyDescriptors
